# 교실 자리 배치 프로그램 개선점 리스트 📋

이 문서는 현재 프로그램의 개선 가능한 사항들을 체계적으로 정리한 것입니다.

---

## 🔴 높은 우선순위 (Critical)

### 1. 코드 구조 및 유지보수성
- [ ] **MainController.ts 파일 크기 문제** (약 8,000줄)
  - 현재 하나의 파일에 모든 로직이 집중되어 있음
  - 리팩토링 필요: 기능별로 더 작은 모듈로 분리
  - 예: `HistoryManager`, `FixedSeatManager`, `LayoutRenderer` 등으로 분리

- [ ] **타입 안전성 강화**
  - `require()` 사용 부분을 ES6 import로 변경 (`RandomService.ts`, `InputService.ts`)
  - 동적 import 대신 정적 import 사용
  - 예: `const { StudentModel } = require('../models/Student')` → `import { StudentModel } from '../models/Student.js'`

- [ ] **에러 처리 일관성**
  - 일부 메서드에서 try-catch가 없거나 불완전함
  - 전역 에러 핸들러와 로컬 에러 처리의 일관성 확보
  - 사용자에게 더 명확한 에러 메시지 제공

### 2. 사용자 경험 (UX)
- [ ] **로딩 상태 표시 부재**
  - 대용량 데이터 처리 시 (100명 이상) 로딩 인디케이터 없음
  - 자리 배치 중 사용자 피드백 부족
  - 제안: 스피너 또는 프로그레스 바 추가

- [ ] **입력 검증 피드백 개선**
  - 실시간 검증 메시지 표시
  - 잘못된 입력 필드에 시각적 하이라이트
  - 예: 빨간 테두리, 에러 아이콘 표시

- [ ] **모바일 반응형 개선**
  - 작은 화면에서 사이드바가 너무 많은 공간 차지
  - 터치 제스처 최적화 (드래그 앤 드롭)
  - 모바일에서 테이블 스크롤 개선

### 3. 접근성 (Accessibility)
- [ ] **키보드 네비게이션**
  - 모든 기능을 키보드만으로 사용 가능하도록 개선
  - Tab 순서 최적화
  - 포커스 표시 명확화

- [ ] **스크린 리더 지원**
  - ARIA 레이블 추가
  - 의미론적 HTML 태그 사용 (`<button>`, `<nav>` 등)
  - 상태 변경 시 알림 제공

- [ ] **색상 대비**
  - WCAG 2.1 AA 기준 충족 확인
  - 색상만으로 정보를 전달하지 않도록 개선

## 🟡 중간 우선순위 (Important)

### 4. 기능 개선
- [ ] **준비중 기능 완성**
  - 모둠 배치 기능 구현 (UI에 표시되어 있으나 미구현)
  - ㄷ자 2명 짝꿍 배치 기능 구현

- [ ] **자리 배치 히스토리 관리 강화**
  - 현재 되돌리기(Ctrl+Z) 기능이 있으나 제한적
  - 히스토리 목록 시각화
  - 특정 시점으로 되돌리기 기능

- [ ] **고정 좌석 기능 개선**
  - 고정 좌석 일괄 지정/해제 기능
  - 고정 좌석 템플릿 저장/불러오기
  - 고정 좌석 시각적 구분 강화

- [ ] **엑셀/CSV 가져오기 개선**
  - 현재 CSV만 지원, 엑셀(.xlsx) 직접 지원
  - 파일 형식 자동 감지
  - 업로드 시 미리보기 기능

### 5. 성능 최적화
- [ ] **대용량 데이터 처리**
  - 100명 이상 학생 처리 시 성능 저하 가능성
  - 가상 스크롤링 구현 검토
  - 렌더링 최적화 (requestAnimationFrame 사용)

- [ ] **메모리 관리**
  - 이벤트 리스너 정리 확인
  - DOM 요소 캐싱 최적화
  - 불필요한 리렌더링 방지

- [ ] **로컬 스토리지 최적화**
  - 데이터 압축 (큰 배치 데이터 저장 시)
  - 저장 용량 제한 관리
  - 오래된 데이터 자동 정리

### 6. 데이터 검증 및 안정성
- [ ] **엣지 케이스 처리**
  - 학생 수가 0인 경우
  - 남학생만 또는 여학생만 있는 경우
  - 좌석 수보다 학생 수가 많은/적은 경우
  - 고정 좌석 수가 학생 수보다 많은 경우

- [ ] **데이터 무결성 검증**
  - localStorage 데이터 손상 시 복구 메커니즘
  - 잘못된 CSV 파일 업로드 시 상세한 에러 메시지
  - 파일 크기 제한 및 검증

- [ ] **입력 검증 강화**
  - 음수 입력 방지 (현재 HTML min 속성만 사용)
  - 매우 큰 숫자 입력 제한
  - 특수문자 포함 이름 처리
